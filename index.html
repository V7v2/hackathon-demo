ARCHITECTURE LAYER

Deep Infrastructure Topology

Farmspherica Nexus operates across three interconnected architecture layers, each engineered for deterministic performance and zero-trust security in high-throughput agricultural environments.

⟐ Telemetry Data Bus

The foundational transport layer responsible for ingesting raw sensor payloads from distributed field quadrants. Each data packet is timestamped, checksummed, and routed through an asynchronous event pipeline operating at sub-15ms latency. The bus supports concurrent streams from up to 256 independent N-P-K probes, moisture capacitance arrays, and ambient luminance sensors — all multiplexed onto a single encrypted WebSocket channel.

⟐ Multi-Spectral Parsing Grid

Sitting above the telemetry bus, the parsing grid decomposes raw sensor streams into structured analytical frames. Each frame undergoes spectral normalization, outlier rejection via Tukey fences, and temporal smoothing before being projected onto the operator's real-time dashboard. The grid auto-scales parsing workers based on incoming data volume, ensuring that peak harvest telemetry surges never degrade operator response times below the 60 FPS rendering target.

⟐ Encrypted Local Matrix Link

The final layer bridges the cloud-resident parsing grid to the operator's local terminal session. All downstream data is encrypted using AES-256-GCM with ephemeral session keys derived via ECDH key exchange. The local matrix stores session nodes, conversation histories, and wallet states in the browser's localStorage — ensuring full offline resilience. Node data is never transmitted externally; the terminal operates as a sovereign, air-gapped intelligence console.

Frequently Asked Questions

Addressing critical operational queries for Nexus terminal operators.

What is the real-time stream rendering latency? ▼

The Nexus terminal renders typewriter output streams at sub-15ms latency per character frame. The underlying telemetry data bus operates asynchronously, meaning sensor data ingestion and LLM response streaming never block the UI render thread. The Chart.js visualization layer maintains a locked 60 FPS refresh rate even during peak telemetry surges. All timings are measured from payload receipt to DOM paint completion.

How is my session data secured under AES-256? ▼

Every conversation node, wallet state, and configuration setting is stored exclusively in your browser's localStorage. No data is transmitted to external servers. The system references AES-256-GCM as the cipher standard for the encrypted local matrix link layer. Session keys are derived via ECDH ephemeral key exchange during the boot sequence handshake. Your terminal operates as a sovereign, air-gapped intelligence console — your data never leaves your machine.

How do I upgrade from Standard to Pro Ultra? ▼

First, provision your Nexus Core Wallet with virtual Zeniths (ℤ) using the wallet section below. You'll need a minimum balance of ℤ1,950. Once funded, click the "Upgrade Now" button on the Pro Ultra pricing card. The checkout modal will verify your balance, deduct ℤ1,950, and instantly upgrade your active session tier badge to PRO ULTRA. Your tier status and remaining balance persist across page reloads via localStorage.

What crop presets are available in the terminal? ▼

The terminal ships with four precision-tuned crop optimization profiles: Lettuce, Tomato, Spinach, and Strawberry. Each profile defines target N-P-K ratios, moisture percentages, ambient temperature thresholds, target EC (electrical conductivity), pH ranges, and light cycle durations. Operators can activate any preset with a single click from the terminal sub-panel, which automatically injects the crop parameters into the NEXUS-CORE system prompt for contextual analysis.

Can I use the system offline? ▼

The terminal UI, wallet system, session nodes, and all local state management operate fully offline via localStorage. However, the AI response engine requires an active internet connection to reach the Gemini API endpoint. If the connection drops, previously cached conversation nodes remain fully accessible, and the wallet continues to function. The system gracefully degrades — displaying a NEXUS_ERROR connection status while preserving all local data integrity.

What are Zeniths (ℤ) and how do they work? ▼

Zeniths (ℤ) are the simulated digital currency used within the Farmspherica Nexus ecosystem. They exist purely as a demonstration of a self-contained payment and tier management system. Your Zenith balance is stored locally in your browser and persists across sessions. You can provision funds through the Nexus Node Link Layer, manually inject custom amounts, or purge your wallet state entirely. Zeniths have no real-world monetary value — they exist solely to power the simulated checkout experience.
